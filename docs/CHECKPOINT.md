# Чекпоинт: Умный реверсивный коридор

## Концепция

**Проблема:** На ремонтных участках дорог (0.5–2 км) реверсивное движение управляется по таймеру (3 мин туда, 3 обратно). Это приводит к:
- Простою одной стороны при асимметричном потоке
- Лобовым столкновениям, когда фура не успела выехать из зоны
- Невозможности дать приоритет экстренным службам

**Решение:** Адаптивная система управления фазами на основе компьютерного зрения. Камеры на въездах/выездах отслеживают каждое ТС (YOLOv8 + ByteTrack). Система гарантирует:
1. Смена фазы **только** когда зона пуста (`count_in == count_out`)
2. Длительность зелёного адаптируется под реальный спрос (приоритет = очередь × w1 + время ожидания × w2)
3. Обнаружение застреваний (ТС в зоне >30с → блокировка смены)
4. Ручной override для экстренных служб

**Главная метрика:** сравнение fixed (3+3 мин) vs adaptive — снижение средней задержки на 30–50%, unsafe switches = 0.

## Архитектура

```
┌─────────┐    MQTT     ┌───────────┐   WebSocket   ┌──────────┐
│  CV     │───────────▶│  Control  │──────────────▶│   Web    │
│ (YOLO)  │            │  (FSM +   │               │(Dashboard)│
│         │◀───────────│  Policy)  │◀──────────────│          │
└─────────┘  cmd/scene  └───────────┘   REST API    └──────────┘
                              │
                    ┌─────────┴─────────┐
                    │  InfluxDB + SQLite │
                    └───────────────────┘
```

**Стек:** Python 3.11 (FastAPI, paho-mqtt), React 19 + TypeScript + Vite + Tailwind, MQTT (Mosquitto), InfluxDB, Docker Compose.

## Что работает сейчас

### Backend (services/control/) — ГОТОВ
- [x] FSM фаз: GREEN_A → ALL_RED_A_to_B → GREEN_B → ALL_RED_B_to_A (цикл)
- [x] Безопасность: count_in == count_out, stuck detection (>180с), comm loss watchdog (5с)
- [x] Адаптивная политика: T_min=15с, T_max=300с, приоритет по очереди + ожиданию
- [x] Режимы: fixed (3+3 мин) и adaptive, переключение через API
- [x] Override: FORCE_GREEN_A/B, ALL_RED, EMERGENCY
- [x] WebSocket-мост MQTT → браузер
- [x] REST API: /scenario/start, /override, /metrics/compare, /health
- [x] Хранение: SQLite (события, фазы, сессии) + InfluxDB (метрики)
- [x] Тесты: 72 теста (FSM, safety, policy, API) — все зелёные

### Frontend (web/) — ГОТОВ
- [x] Real-time дашборд с WebSocket-подключением к бэку
- [x] Визуализация дороги (Canvas): светофоры, машины, ремонтная зона, анимации
- [x] Phase Timeline: лента фаз за 5 минут с легендой
- [x] Queue Gauges: прогресс-бары с цветовыми порогами (зелёный/жёлтый/красный)
- [x] Metrics Panel: KPI-карточки fixed vs adaptive с дельтой, bar chart
- [x] Event Log: группировка дубликатов, иконки по уровню
- [x] Control Panel: выбор сценария, режим, override-кнопки
- [x] Demo-режим: автоматическая генерация данных когда бэк не подключён
- [x] Презентационный режим (клавиша F)
- [x] Тесты: 12 тестов (store, компоненты) — все зелёные

### Инфраструктура — ГОТОВА
- [x] Docker Compose: Mosquitto + InfluxDB + Control service
- [x] Vite dev server с proxy на бэк
- [x] MQTT-контракты зафиксированы (contracts/)

### CV (services/cv/) — НЕ НАЧАТ

## Риски

1. **CV не готов** — бэк и фронт работают через replay.py / demo-режим, но для полной демонстрации нужен реальный видеопоток
2. **Метрики нулевые без сценария** — нужен автозапуск сценария или более наглядный idle-стейт
3. **Нет e2e теста** — replay.py → control → web пока не автоматизирован
